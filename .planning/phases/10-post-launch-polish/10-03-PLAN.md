<objective>
Implement 2FA/MFA for admin accounts using Supabase's native TOTP MFA APIs. Owner accounts are required to enroll on first login; baker accounts can optionally enroll. The pattern uses AAL (Authenticator Assurance Level) enforcement — Supabase tracks whether a session has passed MFA. Recovery is via enrolling a second TOTP factor (Supabase does not support backup codes).

**Decision deviation note:** CONTEXT.md specifies "Recovery: Backup codes generated at setup time — owner saves them." Supabase Auth does not natively support backup codes as a recovery mechanism (there is no `supabase.auth.mfa.generateBackupCodes()` API). The correct recovery mechanism within Supabase's TOTP implementation is to enroll a second TOTP factor on a different device/app. This plan implements that substitute: EnrollMFA.tsx includes a "Enroll Second Device" section that initiates a second `supabase.auth.mfa.enroll()` call for backup access. This is the Supabase-documented recovery pattern and is functionally equivalent to backup codes for this use case.

Purpose: Admin accounts (owner@elisbakery.com, orders@elisbakery.com) handle financial orders and customer data. TOTP 2FA prevents unauthorized access if passwords are compromised.
Output: 3 new auth components; OwnerDashboard and FrontDesk wrapped with AAL enforcement.
</objective>

// Enroll (step 1)
const { data, error } = await supabase.auth.mfa.enroll({ factorType: 'totp' });
// data.id — factor ID for subsequent calls

// data.totp.uri — otpauth:// URI for manual entry

// Challenge (step 2, after enroll OR for already-enrolled users)
const { data: challengeData, error } = await supabase.auth.mfa.challenge({ factorId });
// challengeData.id — challenge ID for verify call

// Verify (step 3)
const { error } = await supabase.auth.mfa.verify({ factorId, challengeId: challengeData.id, code: sixDigitCode });

// Check AAL (post-login enforcement)
const { data } = await supabase.auth.mfa.getAuthenticatorAssuranceLevel();
// data.currentLevel: 'aal1' | 'aal2'
// data.nextLevel: 'aal1' | 'aal2'
// Show MFA when: data.nextLevel === 'aal2' && data.currentLevel !== 'aal2'

// List enrolled factors
const { data: factors } = await supabase.auth.mfa.listFactors();
// factors.totp[0] — first enrolled TOTP factor

// After verify, refresh session so AAL updates immediately
await supabase.auth.refreshSession();
