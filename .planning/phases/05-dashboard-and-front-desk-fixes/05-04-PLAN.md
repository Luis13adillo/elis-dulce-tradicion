**Update the `handleSubmit` function** to show a warning if the new value is lower than today's order count. The `handleSubmit` function already calls `updateMutation.mutateAsync(formData)` — update the success toast to be specific about capacity when that field changed:

```typescript
const handleSubmit = async (e: React.FormEvent) => {
  e.preventDefault();

  // Non-blocking warning if capacity is set very low
  if (formData.max_daily_capacity !== undefined && formData.max_daily_capacity < 1) {
    toast.warning('Capacity must be at least 1. Setting to 1.');
    setFormData({ ...formData, max_daily_capacity: 1 });
    return;
  }

  try {
    await updateMutation.mutateAsync(formData);
    // Check if capacity specifically changed to show targeted success message
    const capacityChanged = settings?.max_daily_capacity !== formData.max_daily_capacity;
    if (capacityChanged) {
      toast.success('Daily capacity updated');
    } else {
      toast.success(
        isSpanish ? 'Configuración guardada exitosamente' : 'Settings saved successfully'
      );
    }
  } catch (error) {
    toast.error(
      isSpanish ? 'Error al guardar configuración' : 'Error saving settings'
    );
  }
};
```

Note: The component already has a form-level Save button that submits the entire form. The capacity field participates in the same form — no separate Save button needed per field. The `formData` state already holds all fields; just adding this field to the JSX is sufficient.
