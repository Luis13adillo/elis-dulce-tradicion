# Plan 09-04: Order.tsx Refactor â€” Extract Step Components + Migrate Delivery into ContactStep

## Goal

Split `src/pages/Order.tsx` (~1,100 lines after 09-03) into 5 step components under `src/components/order/steps/`. Parent `Order.tsx` holds all state and passes props + callbacks. Each step owns its own validation. Replace decorative step progress bars with a clickable step indicator that shows per-step summaries. Delivery UI (wired in 09-03 directly in `Order.tsx`) migrates into `ContactStep.tsx`.

**Depends on 09-03** because the delivery state (`deliveryAddress`, `deliveryFee`, `handleAddressChange`) must already exist in `Order.tsx` before `ContactStep` can receive it as props.
